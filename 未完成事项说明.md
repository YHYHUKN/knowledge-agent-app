### 替换示例代码

只需实现 `synthesizeAnswer` 内部，将规则拼接替换为真实 LLM API 调用：

```typescript
// lib/agent.ts — synthesizeAnswer 改造示例（伪代码）
async function synthesizeAnswer(
  query: string,
  results: SearchResult[]
): Promise<{ answer: string; grounded: boolean }> {
  // 1. 构造上下文
  const context = results.map(r => `[${r.title}]: ${r.content}`).join("\n");

  // 2. 调用 LLM
  const response = await openai.chat.completions.create({
    model: "gpt-4o",
    messages: [
      {
        role: "system",
        content: "你是一个企业知识库问答助手。请仅根据提供的上下文回答问题。回答时必须引用来源，未提及的内容一律说不知道，禁止编造。"
      },
      {
        role: "user",
        content: `上下文：\n${context}\n\n问题：${query}`
      }
    ],
    temperature: 0.3, // 偏低，输出更稳定
    max_tokens: 800
  });

  const answer = response.choices[0].message.content ?? "";
  return { answer, grounded: true };
}
```

### 推荐接入方案

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| OpenAI GPT-4o / GPT-4o-mini | 通用能力强 | 快速验证，推荐先用 |
| 阿里通义千问（Qwen-Max） | 中文优化，支持长上下文 | 国内部署场景 |
| 字节豆包 | 成本低 | 大规模问答 |
| 本地开源模型（Qwen2 / GLM4） | 数据不出境 | 合规敏感企业 |

> **注意**：接入真实 LLM 后，`answer` 内容不再受规则拼接逻辑控制，`grounded` 字段的值应由模型返回的置信度或引用数量来决定，前端 UI 无需改动。
