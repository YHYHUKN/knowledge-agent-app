# 继续迭代优化方向

本文档列出项目从当前"功能可用"状态到"生产就绪"的完整迭代路线图，分为三个阶段。

---

## 第一阶段：工程能力补强（低优先级）

在真实场景上线前必须补齐的基础能力。

### 1. 真实 LLM 对接

- **改动文件**：`lib/agent.ts`（`synthesizeAnswer` 函数）
- **说明**：将规则拼接替换为 OpenAI / 通义千问等真实 LLM API，支持流式输出（Streaming）
- **参考**：详见 `未完成事项说明.md`

### 2. 持久化升级

- **改动文件**：`lib/data.ts`、`lib/client-storage.ts`
- **当前**：服务端内存存储，服务重启后种子数据外内容清空
- **目标**：替换为 SQLite（轻量）或 PostgreSQL（生产）
- **方案**：
  - SQLite：`better-sqlite3` + 磁盘文件，单进程写入
  - PostgreSQL：`pg` 连接池，支持多进程并发
- **注意事项**：需要同步迁移 `lib/client-storage.ts` 的 localStorage 逻辑，改为 API 请求读写数据库

### 3. 资产删除与编辑

- **改动文件**：`app/api/assets/route.ts` + `asset-list.tsx` + `asset-form-dialog.tsx`
- **当前**：仅支持新增（POST），不支持删除（DELETE）和编辑（PUT）
- **目标**：列表增加操作按钮（编辑 / 删除），弹窗支持编辑回填

### 4. 前端搜索过滤

- **改动文件**：`asset-list.tsx`
- **当前**：资产列表仅展示，无过滤能力
- **目标**：支持按标签筛选 / 关键词搜索，实时过滤列表

### 5. Trace 结果持久化

- **改动文件**：API Route 新增 `app/api/traces/route.ts`
- **目标**：每次 AgentTrace 不仅返回前端展示，同时写入数据库，支持后台审计历史记录

## 第二阶段：RAG 能力升级（中等优先级）

从"关键词检索"升级为"语义检索"，提升问答召回质量。

### 1. 向量检索接入

- **改动文件**：`lib/search.ts`、`lib/data.ts`
- **方案**：接入 pgvector（PostgreSQL 扩展）/ Milvus / Pinecone
- **改造要点**：
  - 资产新增时同步生成向量并写入向量库
  - 检索时 query 先过 embedding 模型，再调用向量库 Top-K 召回
  - 向量分数映射为 `SearchResult.score`，接口不变
- **文档 chunk 切分**：长文本按 300~500 token 滑窗切分，一条资产对应多条向量

### 2. 混合检索

- **改动文件**：`lib/search.ts`
- **说明**：关键词检索 + 向量检索融合打分，对精确术语 / 编号 / 代码片段使用关键词权重，对语义相似表达使用向量权重
- **典型算法**：RRF（Reciprocal Rank Fusion）

### 3. 重排序（ReRank）

- **改动文件**：`lib/search.ts`
- **说明**：粗排（向量 Top-K）→ 精排（ReRank 模型，如 bge-reranker-v2.5）→ 最终结果
- **效果**：显著提升精准度，尤其对语义相近但字面不同的表达

### 4. 引用精度提升

- **当前**：引用基于资产级别（整条资产作为 context）
- **目标**：引用精确到 chunk 级别（召回的最相关段落），提升回答可追溯性
- **改动文件**：`lib/search.ts`、`lib/agent.ts`、`trace-panel.tsx`

## 第三阶段：ToB 产品化（高优先级）

面向真实企业客户的完整产品能力。

### 1. 多租户隔离

- **数据层**：`KnowledgeAsset` 增加 `tenantId` 字段，所有查询强制带 `tenantId` 过滤
- **身份与鉴权**：JWT / Session 认证，从 token 解出 `tenantId` 和 `userId`
- **向量库隔离**：带 `tenantId` metadata filter 检索，或物理隔离 collection/namespace
- **改动文件**：核心类型 + 全部 API Route + 前端路由守卫

### 2. 权限分级控制

- **角色**：管理员 / 普通用户 / 只读访客
- **权限矩阵**：

| 操作 | 管理员 | 普通用户 | 只读访客 |
|------|--------|---------|---------|
| 查看资产列表 | ✅ | ✅ | ✅ |
| Agent 问答 | ✅ | ✅ | ✅ |
| 新增资产 | ✅ | ✅ | ❌ |
| 编辑资产 | ✅ | 仅自己创建 | ❌ |
| 删除资产 | ✅ | 仅自己创建 | ❌ |
| 查看 Trace 审计 | ✅ | 仅自己 | ❌ |

### 3. 运营统计仪表盘

- **指标**：问答量趋势 / 召回率 / 拒答率 / 平均响应延迟 / Top 热门问题
- **改动文件**：新增 API Route（统计聚合）+ 新增统计页面组件
- **可视化**：Recharts 折线图 / 柱状图

### 4. 内容审核与合规

- **输入审核**：用户提问敏感词过滤，涉政 / 涉黄 / 涉暴实时拦截
- **输出审核**：LLM 生成内容敏感词检测，合规检查后才返回前端
- **改动文件**：`app/api/agent/chat/route.ts`（前后各加一层审核 hook）

### 5. 缓存与降级

- **Query 缓存**：相同 / 相近问题复用最近结果，降低 LLM 调用成本
- **超时降级**：检索或生成超时时返回"服务繁忙，请稍后再试"，避免前端长时间等待
- **限流**：单用户 / 单租户调用频率限制，防止资源滥用

---

## 迭代优先级建议

```
P0（上线前必须）→ P1（近期迭代）→ P2（中长期规划）
```

| 优先级 | 事项 |
|--------|------|
| **P0** | 真实 LLM 对接、持久化升级、Trace 持久化 |
| **P1** | 向量检索接入、资产删除编辑、权限分级 |
| **P2** | 混合检索、ReRank、运营仪表盘、内容审核、缓存降级 |
